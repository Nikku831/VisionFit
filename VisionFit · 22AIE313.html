Motivation

Objectives

Goal

Develop an automated system that can detect, count, and score four physical activities from video using classical computer vision techniques.

Given a video sequence V = {I₁, … , Iₜ}, where Iₜ ∈ ℝH×W×3, design a computer vision pipeline that automatically analyzes the video frames and identifies activity patterns from recorded exercise videos.

  • Use a static smartphone camera to capture activity videos
  • Processes recorded exercise videos and generates annotated outputs with detected repetitions and metrics.
  • Detect and count four predefined activities from motion patterns
  • Generate annotated video outputs and structured CSV logs
  • Ensure the system is explainable and interpretable
  • Implement the entire pipeline using classical computer vision without trained models

Core CV — Steps 1 & 2

Background Subtraction & Morphology

MOG2 — Gaussian Mixture Model

\[P(x_t)=\sum_{k=1}^{K}w_k\cdot\mathcal{N}(\mu_k,\Sigma_k)\]

Each pixel modeled as \(K\) Gaussians with weights \(w_k\), mean \(\mu_k\), variance \(\Sigma_k\). Foreground if:

\[|I_t-\mu_k|>\lambda\sigma_k\quad\text{for all background Gaussians}\]
  • Adapts dynamically per-frame — handles gradual lighting changes
  • Suitable for static camera setup with moving subject
  • Shadow detection enabled; history window = 500 frames

Morphological Post-Processing

  • Opening \((M\ominus K)\oplus K\) — erosion then dilation; removes isolated noise blobs smaller than kernel
  • Closing \((M\oplus K)\ominus K\) — dilation then erosion; fills small holes inside the silhouette
  • Dilation — expands foreground boundary by 2–3 iterations for clean halo
Kernel: 5×5 elliptical structuring element; ellipse shape matches natural body curvature better than rectangular kernels.
Limitation: MOG2 absorbs a stationary subject over ~30s — critical for Flamingo/Squat sustained postures. Mitigation: lower learning rate or switch to static threshold for those modules.
Core CV — Steps 3 & 4

Contour Extraction & ROI-Based Marker Detection

Contour Extraction

External contours extracted from the binary morphological mask using cv2.findContours(). Largest contour by area = human body. Minimum area gate: 2000 px² rejects noise.

Geometric Features from Contour

  • Bounding box \((x,y,w,h)\) — height \(h_{bb}\), width \(w_{bb}\) used for ratio signals
  • Centroid via image moments: \(c_x=M_{10}/M_{00},\;c_y=M_{01}/M_{00}\)
  • Top-most point \(\arg\min_y\,C\) — head proxy for Flamingo/Jump
  • Bottom-most point \(\arg\max_y\,C\) — foot proxy for all modules
  • Silhouette area \(A=M_{00}\) — depth proxy and instability indicator

ROI-Based Marker Extraction (Squat & Jump)

Manual rectangular ROIs are defined for specific body joints. Each ROI \(R=(x,y,w,h)\) is saved as JSON for reuse across sessions.

Inside each ROI: Shi–Tomasi detects ~80 corners → Lucas–Kanade tracks them across frames → centroid \((\bar{x}_i,\bar{y}_i)\) = joint position.
Core CV — Step 5

Sparse Optical Flow — Shi–Tomasi + Lucas–Kanade

Shi–Tomasi Corner Detection

Detects strong, trackable corners inside each ROI using the structure tensor:

\[M=\sum_{\text{window}}\begin{bmatrix}I_x^2&I_xI_y\\I_xI_y&I_y^2\end{bmatrix},\quad R=\min(\lambda_1,\lambda_2)>\tau\]
  • Point accepted only when gradient is strong in both directions — ensures trackability
  • \(\tau\) tuned per-ROI; ~80 features selected per joint region
  • Features re-initialised every N frames if count drops below threshold

Joint Position from Features

\[(x_c,y_c)=\left(\frac{1}{K}\sum x_i,\;\frac{1}{K}\sum y_i\right),\quad\sigma^2_{\text{mean}}=\frac{\sigma^2}{K}\]

Averaging \(K\) features reduces positional noise variance by factor \(K\).

Pyramidal Lucas–Kanade Flow

Brightness constancy assumption gives the optical flow constraint:

\[I(x,y,t)=I(x+u,y+v,t+1)\;\Rightarrow\;I_xu+I_yv+I_t=0\]

Solved in local window via least squares:

\[(u,v)=(A^TA)^{-1}A^Tb,\quad A=\begin{bmatrix}I_x&I_y\end{bmatrix}\]

Pyramid Strategy

  • Level 0: full resolution; Level 1–2: successively downsampled
  • Coarse-to-fine search handles motions larger than a single window
  • Prevents local minima; improves convergence on fast movements
cv2.calcOpticalFlowPyrLK() with winSize=(15,15), maxLevel=2, criteria: EPS + COUNT.

Squat — ROI Markers & Biomechanical Detection

ROI Marker Points Extracted

Hip Centroid (ROI-Hip)Shi–Tomasi + LK inside hip ROI. ~80 corner features tracked; centroid = \((\bar{x},\bar{y})\). Primary signal anchor.
Ankle Centroids L+R (ROI-Ankle)Two independent ROIs, each tracked separately. Average \(y\) used as single ankle reference for \(D_t\).
Palm Centroids L+R (ROI-Palm)Tracked for skeleton annotation overlay. Not used directly in decision making.
Derived: Neck PointGeometric estimate: Neck_y = Head_y + 0.2×(Hip_y − Head_y). Used for skeleton line drawing.

Biomechanical Signal

\[D_t=\text{Ankle}_y-\text{Hip}_y\qquad D_{\text{smooth}}=\tfrac{1}{7}\textstyle\sum_{i=0}^{6}D_{t-i}\]
\[D_{\text{baseline}}=\text{mean}(D_{\text{first fps frames}})\]
\[\text{Compression Ratio}=\frac{D_{\text{baseline}}-D_{\text{smooth}}}{D_{\text{baseline}}}\]

Calibration on first 1 second establishes standing leg length — makes ratio subject-size and camera-distance independent.

FSM + Cooldown

TransitionConditionEffect
UP → DOWNRatio > 0.15Enter squat
DOWN → UPRatio < 0.06Count +1
CooldownΔframe > 0.7 × fpsBlock double-count

CSV Output per Frame

  • Frame ID, Head \((x,y)\), Neck \((x,y)\), Hip \((x,y)\), Palm \((x,y)\), Ankle \((x,y)\)
  • Compression ratio, squat state, count
Advantage: Ratio is relative — valid for any subject height or camera distance.
Vertical Jump · K Prerana

Jump — ROI Markers, Calibration & Peak Detection

ROI Marker Points Extracted

Head Centroid (ROI-Head)LK optical flow inside head ROI. \(y_{\text{head}}\) provides the top-anchor for body-height calibration and body height consistency check.
Ankle Centroids L+R (ROI-Ankle)Primary signal source. \(y_{\text{ankle}}\) tracked per-frame. Jump = upward ankle displacement. Average of L+R used.
Derived: Body Height \(H_{\text{px}}(t)\)H_px = y_ankle − y_head per frame. Median \(H_{\text{median}}\) gives pixel-to-cm ratio. Variance of \(H_{\text{px}}\) used as tracking quality metric.

Pixel-to-cm Calibration

\[\text{px\_to\_cm}=\frac{H_{\text{real}}}{H_{\text{median}}},\quad H_{\text{median}}=\text{median}(H_{\text{px}})\]
\[\text{variation}=\frac{\sigma(H_{\text{px}})}{H_{\text{median}}}\times100\quad\text{(quality check)}\]

Savitzky–Golay Smoothing

Fits local polynomial — preserves peak shape unlike moving average:

\[\min\sum_{i=-k}^{k}(y_{t+i}-P(i))^2,\quad P(i)=a_0+a_1i+a_2i^2\]

Peak Detection & Height

  • Detect peaks in −ankle_y (upward jump = y decreasing)
  • find_peaks: prominence ≥ 5, distance ≥ 5 frames
  • Baseline = 80th percentile of smoothed \(\text{ankle}_y\)
\[h_{\text{cm}}=(y_{\text{baseline}}-y_{\text{peak}})\times\text{px\_to\_cm}\]

Batch Output

  • Per-jump: air-time, height (cm), frame range
  • Summary: mean, max, min jump height across all videos
Limitation: Stationary camera, known body height, full-body visibility required throughout.

Flamingo — Silhouette Markers & Postural Sway

Marker Points Extracted from Contour

Silhouette Centroid \((c_x, c_y)\)Image moments from largest contour: M = cv2.moments(cnt), c_x=M10/M00, c_y=M01/M00. Primary sway signal — no ROI needed.
Centroid Displacement PathEuclidean shift \(\text{Shift}_t = \|c_t - c_{t-1}\|_2\) forms a time-series sway signal. EMA applied for smoothing.
Bounding Box Bottom\(\arg\max_y\,\text{contour}\) — foot position. Sudden large drop indicates balance loss or subject falling out of frame.
Silhouette Area \(A = M_{00}\)Large sudden area change (e.g. >40%) triggers instability flag independent of centroid.

MOG2 for Foreground + Morphology

Unlike Squat/Jump, Flamingo uses MOG2 for full-body segmentation. Minimum contour area filter: 2000 px². Morphological kernel: 5×5 ellipse (open + close).

Sway Signal Chain

\[\text{Shift}_t=\sqrt{(c_x^t-c_x^{t-1})^2+(c_y^t-c_y^{t-1})^2}\]
\[S_t=\alpha S_{t-1}+(1-\alpha)X_t,\quad\alpha=0.8\quad\text{(EMA)}\]

Stability Classification

Smoothed ShiftLabelMeaning
< 12 pxSTABLENormal micro-sway
12 – 25 pxBORDERLINEIncreased postural sway
≥ 25 pxINSTABILITY EVENTMajor balance correction
\[\text{Balance Time}=\frac{\text{Stable Frames}}{\text{fps}}\qquad\text{Score}=\frac{\text{Stable Frames}}{\text{Total Frames}}\in[0,1]\]
Centroid sway approximates CoM projection motion — low-cost alternative to clinical force plates and IMU sensors.

Jumping Jack — Tracked Body Points & Limb Spread Signal

Tracked Marker Points (Tracked Body Points)

Wrist Landmarks L+RBody landmarks are estimated using ROI-based feature tracking and centroid estimation from tracked points. Silhouette-based measurements are used for activities such as jumping jacks.
Ankle Landmarks L+R\(D_{\text{legs}}=\|ankle_L-ankle_R\|_2\). Combined with \(D_{\text{arms}}\) into composite spread signal \(S(t)\).
Shoulder MidpointReference point for relative arm spread. Also used to anchor vertical centroid oscillation validation check.
Head + Ankle SpanBody height proxy \(\|ankle-head\|_2\) used to normalize all spread distances — removes camera-distance dependency.

Composite Signal & Normalization

\[S(t)=\frac{D_{\text{arms}}(t)+D_{\text{legs}}(t)}{2}\qquad D_{\text{norm}}=\frac{D}{\|ankle-head\|_2}\]

Signal is periodic: \(S(t)\approx A\sin(\omega t)\). One rep = one CLOSED→OPEN→CLOSED cycle.

Alternative Signals Considered

  • Shoulder angle: \(\theta=\cos^{-1}\!\left(\frac{\vec{AB}\cdot\vec{BC}}{|\vec{AB}||\vec{BC}|}\right)\)
  • Bounding box width ratio — simpler but less precise
  • Knee separation ratio \(D_{\text{legs}}/H_{\text{body}}\)

FSM with Hysteresis

TransitionConditionEffect
CLOSED → OPEN\(S(t)>T_{\text{open}}\)Enter open phase
OPEN → CLOSED\(S(t)<T_{\text{close}}\)Count +1
Cooldown\(t-t_{\text{last}}>\Delta\)Block double-count

\(T_{\text{open}}>T_{\text{close}}\) — asymmetric thresholds prevent oscillation at boundary.

Normalization by body height makes thresholds camera-distance independent across all subjects.

Summary

Conclusion

VisionFit shows that classical computer vision — carefully engineered around meaningful anatomical keypoints via Shi–Tomasi + Lucas–Kanade tracking — can objectively assess fitness activities without deep learning, datasets, or GPU hardware.

Unified Pipeline

All modules share common classical computer vision components (feature tracking, signal analysis, and threshold-based state transitions) while activity-specific logic is implemented separately.

Meaningful Markers

Hip/ankle (squat), head/ankle (jump), centroid (flamingo), wrists/ankles (jack) — all physically grounded

Scale Invariance

Compression ratio, body-height calibration, and normalized limb spread work across subjects and camera distances

Activity-Matched Smoothing

MA, EMA, Savitzky–Golay chosen per the signal type of each module

CPU Real-Time

Lightweight pipeline suitable for CPU execution without deep learning dependencies; no ML runtime dependencies, deployable anywhere

VisionFit is low-cost, explainable, and deployable wherever coaches and sensors are unavailable.
